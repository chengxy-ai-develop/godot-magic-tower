# 贡献指南

欢迎参与《Godot Magic Tower》的开发！

## 🚀 如何贡献

### 1. Fork 仓库

点击 GitHub 页面右上角的 "Fork" 按钮

### 2. 克隆仓库

```bash
git clone https://github.com/你的账号/godot-magic-tower.git
cd godot-magic-tower
```

### 3. 创建分支

```bash
git checkout -b feature/your-feature-name
```

### 4. 进行修改

- 遵循项目的代码规范
- 确保代码可以正常运行
- 添加必要的注释

### 5. 提交更改

```bash
git add .
git commit -m "feat: 添加你的功能描述"
```

### 6. 推送到远程

```bash
git push origin feature/your-feature-name
```

### 7. 创建 Pull Request

在 GitHub 上创建 PR，描述你的更改

---

## 📝 提交信息规范

我们使用约定式提交（Conventional Commits）：

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档更新 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构 |
| `test` | 测试相关 |
| `chore` | 构建/工具/配置 |

示例：
- `feat: 添加战斗预览功能`
- `fix: 修复钥匙门碰撞检测`
- `docs: 更新 README 安装说明`

---

## 🎨 代码规范

### GDScript 规范

- 使用 4 个空格缩进
- 函数名使用 snake_case
- 类名使用 PascalCase
- 常量使用 UPPER_CASE
- 私有变量以下划线开头 `_variable`

### 文件组织

- 场景文件放在 `scenes/` 目录
- 脚本文件放在 `scripts/` 目录
- 资源文件放在 `assets/` 目录
- 数据配置放在 `data/` 目录

---

## 🧪 测试

在提交前请确保：

- [ ] 游戏可以正常启动
- [ ] 核心功能可以正常运行
- [ ] 没有控制台错误
