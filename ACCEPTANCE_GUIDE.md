## 🚀 构建流程

### 步骤 1: 获取代码

```bash
# 进入工作目录
cd /home/chengxy/.openclaw/workspace

# 克隆项目 (如果还没有)
git clone https://github.com/chengxy-ai-develop/godot-magic-tower.git

# 进入项目目录
cd godot-magic-tower

# 查看分支
git branch -a

# 切换到主分支 (魔塔 1.0 完整版)
git checkout main

# 或者切换到框架开发分支
git checkout feature/framework-dev
```

### 步骤 2: 验证项目结构

```bash
# 检查核心文件
ls -la

# 应该看到:
# - project.godot (项目配置)
# - core/ (框架核心)
# - scenes/ (场景文件)
# - scripts/ (脚本文件)
# - data/ (数据文件)
# - assets/ (资源文件)
```

### 步骤 3: 打开 Godot 项目

1. **启动 Godot 4.x**
2. **导入项目:**
   - 点击 "导入" (Import)
   - 浏览到 `godot-magic-tower/project.godot`
   - 点击 "导入并编辑" (Import & Edit)

3. **等待导入完成:**
   - Godot 会自动扫描和导入资源
   - 首次导入可能需要 1-2 分钟

## 🐛 常见问题排查

### 问题 1: Godot 无法导入项目
**症状:** 导入时报错 "Invalid project.godot"
**解决:**
```bash
# 检查 project.godot 文件
cat project.godot

# 确认 Godot 版本 (需要 4.x)
godot --version
```

### 问题 2: 运行时报错 "脚本未找到"
**症状:** 控制台显示 "Failed to load script"
**解决:**
```bash
# 检查脚本文件是否存在
ls -la core/*.gd
ls -la scripts/*.gd

# 重新导入资源
# Godot 菜单：场景 → 重新加载当前场景
```

### 问题 3: 数据文件加载失败
**症状:** "Floor data not found" 或 "Enemy not found"
**解决:**
```bash
# 检查数据文件
ls -la data/*.json

# 验证 JSON 格式
python3 -m json.tool data/floors_1-5_detailed.json
```

### 问题 4: 输入无响应
**症状:** 玩家无法移动
**解决:**
```bash
# 检查输入映射
# Godot 菜单：项目 → 项目设置 → 输入映射

# 确认输入动作存在:
# move_up, move_down, move_left, move_right
```
